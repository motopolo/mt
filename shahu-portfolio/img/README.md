## Image Guidelines

### Quality & Format
- Use high-quality images (avoid pixelation)
- Optimize for web (compress to reduce file size)
- Use modern formats (WebP when possible, fallback to JPG/PNG)
- Project images should showcase the actual interface/results

### Sizing
- Profile photo: Square format, professional headshot
- Project images: Consistent aspect ratio (4:3 recommended)
- Icons: Must be exact pixel dimensions as specified
- Screenshots: Actual browser/device screenshots

### Content
- Ensure all images are your own work or properly licensed
- Project images should represent real projects you've worked on
- Profile photo should be professional and recent
- Screenshots should show the actual portfolio website

## Alternative: Use Placeholder Images

If you don't have the images ready, you can use placeholder services:
- https://via.placeholder.com/
- https://picsum.photos/
- https://unsplash.com/ (for professional photos)

Example placeholder URLs that you can use temporarily:
- Profile: `https://via.placeholder.com/300x300/3b82f6/ffffff?text=Your+Photo`
- Projects: `https://via.placeholder.com/800x600/f3f4f6/6b7280?text=Project+Screenshot`
- Icons: Generate using online PWA icon generators

## Notes

- The portfolio is fully functional without images (will show placeholders)
- Update the `src` attributes in HTML when you add your actual images
- Consider using a CDN for better performance if hosting online
- All image references in the code point to this `./img/` directory
